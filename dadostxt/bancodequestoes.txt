2. Aula 02: Estrutura Básica

Toda página na web, da mais simples à mais complexa, é construída sobre um esqueleto fundamental: a estrutura básica do HTML. Esta aula apresentou os pilares dessa estrutura, começando pela declaração <!DOCTYPE html>, que sinaliza ao navegador a versão do HTML que está sendo usada. Em seguida, a tag <html> age como o contêiner mestre, abraçando duas seções vitais: o <head>, que guarda os "pensamentos" da página (metadados e o título da aba), e o <body>, que contém tudo o que é visível para o usuário. Dominar essa estrutura é o primeiro passo real na codificação de uma página.

3. Aula 03: Meta Tags, Cabeçalho e Parágrafo

Uma página web eficaz comunica em dois níveis: com as máquinas e com as pessoas. As meta tags, inseridas no <head>, são a linguagem secreta que usamos para conversar com navegadores e mecanismos de busca, definindo o conjunto de caracteres (UTF-8), a descrição do site e as palavras-chave para otimizar a indexação. Para os leitores humanos, as tags de cabeçalho (<h1> a <h6>) e de parágrafo (<p>) são os blocos de construção essenciais, criando uma hierarquia visual e lógica que torna o conteúdo claro e legível.

4. Aula 04: Formatação de Texto

Além de estruturar o conteúdo em blocos, o HTML oferece um arsenal de tags para formatar o texto, adicionando ênfase, significado e clareza. Esta aula diferencia a formatação puramente visual, como um simples negrito (<b>), daquela com peso semântico, como a ênfase forte (<strong>), que sinaliza importância para os navegadores. Aprender a usar essas ferramentas permite destacar informações, riscar dados obsoletos e estruturar o texto de maneira rica e expressiva.

5. Aula 05: Elementos de Citação

Para construir uma web confiável e bem referenciada, o HTML fornece um conjunto de tags de citação que adicionam significado semântico ao conteúdo. Elas nos permitem marcar abreviações (<abbr>), fornecer informações de contato (<address>), citar títulos de obras (<cite>), e diferenciar entre citações curtas (<q>) e longas (<blockquote>). O uso correto dessas tags enriquece o documento, melhorando a acessibilidade e a forma como os mecanismos de busca interpretam a informação.

Enquanto as tags de citação comunicam significado ao navegador, os comentários comunicam intenção ao desenvolvedor. A seguir, aprenderemos a deixar notas e explicações no código que são invisíveis para o usuário final, uma prática essencial para a colaboração e a manutenção do código a longo prazo.

6. Aula 06: Comentários

No desenvolvimento de software, o código que não é executado é tão importante quanto o que é. Os comentários em HTML são a ferramenta estratégica para isso: eles permitem deixar notas, explicações e lembretes para futuros desenvolvedores (incluindo você mesmo), sem que nada apareça na página final. Além de documentar o código, eles são indispensáveis para desativar temporariamente blocos de HTML durante testes, uma técnica crucial para depuração.

9. Aula 09: Tabelas

Essas linhas são preenchidas por células: <th> (table header) para os cabeçalhos das colunas, que indicam o tipo de dado, e <td> (table data) para as células de dados que preenchem a tabela.

10. Aula 10: Listas

Para organizar sequências de itens, o HTML oferece dois tipos principais de listas. 13. Aula 13: Áudio

Com o HTML5, incorporar áudio em uma página web tornou-se uma tarefa nativa e direta através da tag <audio>. O sucesso da sua implementação depende de atributos essenciais: controls é indispensável para exibir a interface do player (play, pause, volume), enquanto a tag aninhada <source> especifica o caminho do arquivo de áudio (src) e seu formato. Para um controle mais refinado, atributos como controlslist="nodownload" permitem personalizar a interface, ocultando, por exemplo, o botão de download.

14. Aula 14: Vídeos

15. Aula 15: Div e HTML Semântico

A construção de layouts em HTML evoluiu da onipresente <div> — um contêiner genérico sem significado inerente — para uma abordagem mais inteligente com o HTML Semântico. Tags como <header>, <nav>, <section>, <article> e <footer> não apenas agrupam conteúdo, mas descrevem seu propósito. Adotar essa prática não muda a aparência visual inicial da página, mas melhora drasticamente a acessibilidade para leitores de tela, o SEO para mecanismos de busca e a clareza do código para desenvolvedores, estabelecendo o padrão para a construção de websites robustos e significativos.
